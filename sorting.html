Processing - открытый язык программирования, основанный на Java. Представляет инструментарий для работы с изображениями, анимацией и интерфейсами. Processing.js - это web-фреймворк, использующий синтаксис языка Processing. Коды приведённых в статье программ можно запустить в своём баузере, используя редакторы studio.sketchpad.cc и openprocessing.org

Сортировка числового массива

Метод сортировки обменами лежит в основе некоторых более совершенных алгоритмов, хотя сам практически не применяется вне учебной литературы ссылка

"Алгоритм состоит из повторяющихся проходов по сортируемому массиву. За каждый проход элементы последовательно сравниваются попарно и, если порядок в паре неверный, выполняется перестановка элементов." Замечу, что попарно сравниваются именно соседние элементы, то есть элементы с индексами i и i+1 (буквой i обычно обозначается индекс элемета массива).

Создание массива числовых элементов

Пускай массив состоит из 10 элементов. Пусть каждый элемент имеет порядковый номер. Этот номер и определяет положение элемента в отсортированном массиве. Пускай в неотсортированном массиве положение элемента определяется смещением. Путем изменения этого параметра элемент с большим порядковым номером смещается дальше (от начала координат) элемента с меньшим порядковым номером. Инициализация элементов выполняется в функции setup() 

   void setup() {
size(500, 200);
    mods = new Module[moduleSize];
    mods[0] = new Module( 0,40);
    mods[1] = new Module( 5,80);
    mods[2] = new Module( 3,120);
    mods[3] = new Module( 6,160);
    mods[4] = new Module( 2,200);
    mods[5] = new Module( 4,240);
    mods[6] = new Module( 8,280);
    mods[7] = new Module( 7,320);
    mods[8] = new Module( 9,360);
    mods[9] = new Module( 1,400);
}

Создание класса

Пусть каждый элемент массива является экземпляром класса Module (пример работы с классом по ссылке). Пусть число, определяющее величину элемента, то есть то, сколько элементов массива превосходит данный элемент (максимальный элемент превосходит все остальные элементы), содержится в переменной Hight, а смещение числа от начала координат в переменной xOffset. Пусть метод display() отвечает за отрисовку символа. Одним из базовых принципов ООП является полиморфизм, и данный метод можно отнести к полиморфным, т.к. метод может отображать величину как в виде текста, так и виде прямоугольника, в котором данная величина является высотой. Можно провести аналогию между классом и неким работником, выполняющим возложенные на него поручения. Работник обладает ограниченной памятью, он помнит, какое число содержится в элементе, помнит смещение элемента от начала коордитат. Используя метод display() работник может отобразить данное число, сместив его на заданное расстояние.

Код класса Module 

  class Module {
  int Offset;
  int Hight;
  // Contructor
   Module( int tempHight,tempOffset) {
    Hight=tempHight;
    Offset = tempOffset;
    }
  // Custom method for drawing the object
   void display() {
      textSize(20);
      text(Hight,Offset, 70); // Offset, Hight
    }
}

Перемещение ползунка

При нажатии на кнопку mousePressed() элементы сравниваются (меняются местами) попарно. 

 void mousePressed() {
 if(mods[slider].Hight > mods[slider+1].Hight)
        {
        vTemp= mods[slider].Hight;
        mods[slider].Hight=mods[slider+1].Hight;
        mods[slider+1].Hight=vTemp;
        }
}
При отпускании кнопки mouseReleased() инкрементируется переменная, отвечающая за положение ползунка и таким образом ползунок смещается вправо. 
 void mouseReleased() {
    slider++;
    if(slider>=moduleSize-1) {  slider=0; }
  }
Ползунок возвращается в начальную позицию slider=0; при выходе за границы массива.
